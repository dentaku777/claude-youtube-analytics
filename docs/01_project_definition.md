## 1. プロジェクトの背景と目的

### 1.1 背景
YouTube チャンネル分析ツール「つべサーチ」(https://tube-search.xyz) は、既存運営者の自チャンネル可視化に主眼を置いており、**これから動画作成を始める方の参入判断・市場分析には機能が不足している**。
また API キーをローカル保存・認証なしで運用しているため、複数端末利用や安全性に課題がある。

これに対し、本プロジェクトでは以下のニーズを満たす **「新規動画クリエイター向け市場分析 SaaS」** を新規構築する。

- 参入候補ニッチの市場規模・競合状況の可視化
- 競合・ベンチマークチャンネル群（Watchlist）の定点観測
- 「勝ち動画」のパターン抽出（公開時間・尺・サムネ・タイトル特徴）
- 複数端末でのシームレスな利用（API キー・履歴・Watchlist の永続化）
- API キー漏洩リスクの低減（サーバー側暗号化保存）
- 将来の生成 AI 連携（タイトル提案・ニッチレコメンド等）の基盤

### 1.2 目的
**これから動画作成を始める方が「どのニッチで、どんな動画を作れば勝てるか」を意思決定できる** ための市場分析プラットフォーム「**Youtube Analyzer**」を構築する。

### 1.3 提供価値と中心概念

**中心概念: Watchlist（ベンチマーク）**
ユーザーが注視する競合・ベンチマーク的なチャンネル群を Watchlist として登録し、すべての分析機能が Watchlist を横断的に参照する設計とする。これにより：
- 検索 → Watchlist 追加 → 比較 → Insights 横断分析 → 戦略立案 のワンストップ体験
- 同じデータを各機能間で再利用（API Quota 効率化）
- 「自分の市場ポートフォリオ」を継続的に育てる UX

### 1.4 ビジネスゴール
- **MVP リリース**: 約 **18 セッション**（Claude Code でのペアプロを想定。集中して進めれば実時間で 2〜4 週間程度）
- **初期ユーザー獲得**: リリース後 3 ヶ月で 100 アカウント
- **将来の収益化**: フリーミアム or サブスクリプション化（本プロジェクト範囲外）
- **将来の生成 AI 機能**: タイトル提案、ニッチレコメンド、サムネ評価、競合分析サマリ等（フェーズ 2）

> **セッションの定義**: 1 セッション ≒ ユーザーが集中作業できる 2〜3 時間分の Claude Code ペアプロを指す。実時間はユーザーの確認待ち・外部サービス設定・実機テスト等の律速によって変動するため、暦日数ではなくセッション数で見積もる。

## 2. スコープ

### 2.1 対象範囲（In Scope）

#### A. 基本分析機能（オリジナル「つべサーチ」と同等）
- 単一チャンネル分析（`/search`）
  - 8 種のチャンネル KPI、6 種の動画 KPI
  - 期間フィルタ、Shorts／動画区分、ソート、伸び率閾値ハイライト
  - 過去 1 年の投稿頻度・再生数推移グラフ
- 最大 3 チャンネル比較（`/compare`）
- API キー管理（`/settings`）
- CSV エクスポート（12〜13 列、UTF-8 BOM）
- サムネイル ZIP 一括ダウンロード
- 表示列・並び順・表示件数のカスタマイズ（永続化）

#### B. 市場分析機能（本プロジェクト独自）
- **Watchlist 管理（`/watchlist`）** — 最大 30 チャンネル / ユーザー
- **Insights ダッシュボード（`/insights`）** — Watchlist 横断分析
  - 勝ち動画パターン分析（伸び率 ≥ 100% の動画から共通点抽出）
  - 投稿時間ヒートマップ（曜日 × 時刻）
  - サムネギャラリー（高伸び率動画のサムネ並列表示）
  - タイトル頻出語分析（kuromoji 形態素解析）
  - 動画尺 × 伸び率分布
  - エンゲージメント拡張 KPI（Likes/Views, Comments/Views）
- **キーワード市場分析（`/keywords`）** — 検索キーワードの動画市場規模・競合数・伸び率分布
- **競合チャンネル発見（`/discover`）** — 入力チャンネルと類似チャンネルを推薦

#### C. 認証・運用機能
- ユーザー認証（メール+パスワード / Google OAuth）
- API キーのサーバー側暗号化保存（AES-256-GCM）
  - YouTube Data API v3 キー
  - 生成 AI API キー（OpenAI / Anthropic / Google Generative AI）の保存スロット
- 検索画面のシンプル化（URL 入力のみで分析開始、キー設定不要の UX）
- 検索履歴（`/history`）
- パスワードリセット、メール認証
- **オリジナルとは異なるデザイン方針（ダーク基調・データ密度高めのアナリティクス UI）**

### 2.2 対象範囲外（Out of Scope）

#### MVP 範囲外（フェーズ 2 以降で検討）
- **生成 AI 機能の本実装**:
  - AI ニッチレコメンド、AI タイトル生成、AI サムネ評価、AI コンテンツギャップ抽出
  - ※ API キー保存スロットは MVP で実装、機能本体はフェーズ 2
- **チャンネル成長軌跡（長期推移）**
- **新着動画通知（定期ジョブ）**
- 動画コメント取得・分析、感情分析
- チーム機能、マルチテナント、権限管理
- 課金・サブスクリプション機能

#### 完全対象外
- YouTube 投稿予約・配信操作機能
- モバイルネイティブアプリ
- 国際化（多言語対応）— 日本語のみ
- リアルタイム通知（Webhook、メール通知等）

## 6. 制約条件

### 6.1 技術的制約
- YouTube Data API v3 の Quota（10,000 ユニット/日/API キー）
  - **`search.list` は 100 ユニット/回と高コスト**（キーワード分析・関連チャンネル発見で利用）
  - Watchlist 30 ch 全件更新で約 600〜2,000 ユニット消費
- Vercel のサーバーレス関数実行時間制限（無料枠：10 秒、Pro：60 秒）
- Neon の無料プラン制限（0.5GB ストレージ、Compute 自動停止）

### 6.2 法的・規約上の制約
- YouTube API Services Terms of Service の遵守
- 個人情報保護法の遵守（個人情報＝メールアドレスの取扱明示）
- ユーザーが入力した API キー（YouTube・生成 AI）は「ユーザー資産」として暗号化保護

### 6.3 リソース制約
- 開発体制は AI ペアプログラミングのみ
- **予算は完全無料枠運用**（Vercel Hobby、Neon Free、Resend Free、GitHub Free 等）

## 7. 前提条件

- ユーザーは Google Cloud Console で YouTube Data API v3 のキーを発行できる
- 開発・運用環境は Windows（開発機）+ Vercel（本番）
- Node.js 20 LTS 以上を利用（現在の開発機は v24.13.0）
- ドメインは Vercel 提供サブドメインを使用（独自ドメインは将来検討）

## 8. リスク

| # | リスク | 影響度 | 対策 |
|---|---|---|---|
| R1 | YouTube API 仕様変更 | 高 | API クライアントを抽象化レイヤで隔離、エラー監視 |
| R2 | API キー漏洩（YouTube / AI 双方） | 高 | サーバー側のみで復号、最小権限、ENCRYPTION_KEY のローテーション手順整備 |
| R3 | Quota 超過（特に search.list 100 ユニット） | **高** | AnalysisCache で重複呼び出し抑制、キーワード分析は明示確認モーダル付き、Quota 残量リアルタイム表示 |
| R4 | オリジナル「つべサーチ」との商標・規約問題 | 中 | プロダクト名を「Youtube Analyzer」に変更、デザインを差別化、機能の独自性を確保（Watchlist / Insights） |
| R5 | スパム・不正利用 | 中 | Rate Limit、メール認証必須、reCAPTCHA（必要時） |
| R6 | 大量動画チャンネルでのパフォーマンス劣化 | 中 | ページネーション + 期間打ち切り + AnalysisCache |
| R7 | 無料枠超過（Vercel/Neon/Resend） | 中 | ユーザー数・送信メール数・DB サイズの監視、必要時にプラン昇格 |
| R8 | Watchlist 30 ch 一斉更新時の関数タイムアウト | 中 | チャンネル単位で個別更新、バッチ並列度を Vercel Hobby 制約に合わせて調整 |
