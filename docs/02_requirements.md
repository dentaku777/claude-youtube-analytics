## 1. 用語定義

| 用語 | 定義 |
|---|---|
| Quota | YouTube Data API v3 が API キーごとに 1 日あたり付与する利用枠（10,000 ユニット）。**毎日 PST 0:00（JST 17:00）にリセット** |
| 伸び率 (Spread Rate) | 動画 1 本の指標。`再生数 ÷ チャンネル登録者数 × 100`（%） |
| 勝ち動画（ヒット動画） | 伸び率 **≥ 100%** の動画。本システムの「成功動画」基準 |
| Shorts | 60 秒以下の縦型動画。`duration ≤ 60s` または `#shorts` ハッシュタグで判定 |
| 通常動画 | 180 秒（3 分）以上の動画 |
| BYOK | Bring Your Own Key。ユーザーが自身で発行した API キーを持ち込んで利用する方式 |
| KPI | Key Performance Indicator |
| AI キースロット | OpenAI / Anthropic / Google Generative AI の API キー保存枠。MVP では保存のみ |
| **Watchlist** | **ユーザーが注視するベンチマークチャンネル群。最大 30 ch / ユーザー** |
| **AnalysisCache** | Watchlist 内チャンネルの分析データを保持するキャッシュ。各画面・機能が横断的に参照 |
| Insights | Watchlist 横断のメタ分析（パターン抽出、ヒートマップ等） |

## 2. 機能要件

### 2.1 認証機能（F-AUTH）

| ID | 機能 | 詳細 |
|---|---|---|
| F-AUTH-01 | サインアップ（メール） | メール+パスワードでアカウント作成。メール認証必須 |
| F-AUTH-02 | サインアップ（Google） | Google OAuth で 1 クリックサインアップ |
| F-AUTH-03 | ログイン | メール+パスワード or Google OAuth |
| F-AUTH-04 | ログアウト | 全セッション破棄 |
| F-AUTH-05 | パスワードリセット | メール経由でリセットリンク送信 |
| F-AUTH-06 | メール認証 | 認証完了まで主要機能利用不可 |
| F-AUTH-07 | アカウント削除 | ユーザー自身による退会（関連データすべて物理削除） |
| F-AUTH-08 | セッション管理 | 30 日間有効、Refresh Token Rotation |

### 2.2 API キー管理機能（F-KEY）

**設計方針**: `/settings` 画面で YouTube・生成 AI 双方のキーを一括管理。検索・比較画面ではキー入力不要。

#### YouTube Data API キー
| ID | 機能 | 詳細 |
|---|---|---|
| F-KEY-01 | YouTube API キー登録 | 1 ユーザー 1 キー。`AIzaSy...` 形式バリデーション |
| F-KEY-02 | YouTube API キー疎通テスト | 登録時に `channels.list` で実際に API を叩いて検証 |
| F-KEY-03 | YouTube API キー更新 | 既存キーを上書き |
| F-KEY-04 | YouTube API キー削除 | キーを削除すると分析機能は利用不可 |
| F-KEY-05 | 暗号化保存 | AES-256-GCM でサーバー側暗号化（DB に暗号文のみ） |
| F-KEY-06 | キー表示マスク | UI 上は `AIzaSy****...****XXXX` 形式でマスク |
| F-KEY-07 | Quota 残量表示 | 当日の Quota 消費目安をユーザーに表示（推定値、PST 0:00 / JST 17:00 リセット明示） |
| F-KEY-08 | キー無効時のチェックリスト表示 | 疎通失敗時に以下を案内：①正確にコピーされているか ②YouTube Data API v3 が有効化されているか ③スペース・改行混入 ④Quota 超過 |

#### 生成 AI API キー（保存スロット）
| ID | 機能 | 詳細 |
|---|---|---|
| F-KEY-10 | OpenAI API キー保存 | `sk-...` 形式バリデーション、AES-256-GCM 暗号化 |
| F-KEY-11 | Anthropic API キー保存 | `sk-ant-...` 形式バリデーション、AES-256-GCM 暗号化 |
| F-KEY-12 | Google Generative AI キー保存 | `AIzaSy...` 形式バリデーション、AES-256-GCM 暗号化 |
| F-KEY-13 | AI キー疎通テスト（任意） | モデル一覧 API のみ叩く（**課金ゼロ厳守**） |
| F-KEY-14 | AI キー削除 | 個別削除 |
| F-KEY-15 | AI キー利用通知 | MVP では「保存のみ。AI 分析機能はフェーズ 2 で提供」と明示 |

### 2.6 **Insights ダッシュボード（F-INSIGHT）**【新規】

**`/insights` で Watchlist 内のデータを横断的に分析。**

| ID | 機能 | 詳細 |
|---|---|---|
| F-INSIGHT-01 | サブセット選択 | Insights 対象とする Watchlist チャンネルをチェックボックスで絞り込み（デフォルト全件） |
| F-INSIGHT-02 | 期間指定 | 分析期間（1 ヶ月 / 3 ヶ月 / 6 ヶ月 / 1 年） |
| **F-INSIGHT-10** | **勝ち動画パターン分析** | 伸び率 ≥ 100% の動画を集計し、公開曜日 / 公開時刻 / 動画尺 / Shorts比率 / タイトル特徴の傾向を可視化 |
| **F-INSIGHT-11** | **投稿時間ヒートマップ** | 全 Watchlist の投稿時刻を曜日 × 時刻のマトリクス（7 × 24）でヒートマップ表示。色濃度＝投稿数 or 平均伸び率（切替） |
| **F-INSIGHT-12** | **サムネギャラリー** | 伸び率 ≥ 100% の動画サムネをグリッド表示（最大 100 件）。クリックで YouTube 遷移。**目視パターン認識用** |
| **F-INSIGHT-13** | **タイトル頻出語分析** | kuromoji 形態素解析で名詞・動詞を抽出 → N-gram (1, 2, 3) 頻出語ランキング。勝ち動画 vs 全動画 で比較表示 |
| **F-INSIGHT-14** | **動画尺 × 伸び率分布** | X軸: 動画尺(分)、Y軸: 伸び率(%) の散布図。Shorts / 通常で色分け。最適な尺帯を特定 |
| **F-INSIGHT-15** | **エンゲージメント拡張 KPI** | Likes/Views (%), Comments/Views (%), バイラル係数 (= 伸び率 × Likes/Views) を表で表示 |
| F-INSIGHT-20 | Insights のエクスポート | 各分析結果を CSV / 画像 (PNG) で書き出し |

### 2.12 将来機能（生成 AI、フェーズ 2 想定）— 参考

MVP では実装しないが、API キー保存スロットと Watchlist データ基盤を用意しておくことで、低摩擦に追加可能。

- AI-01: **AI ニッチレコメンド** — Watchlist + 関心キーワードから「あなたに合うニッチ」提案
- AI-02: **AI タイトル生成** — 勝ち動画パターンを学習データに新作タイトル案を生成
- AI-03: **AI サムネ評価** — GPT-4V / Claude / Gemini でサムネ採点
- AI-04: **AI コンテンツギャップ** — Watchlist 競合動画タイトルから未開拓トピックを抽出
- AI-05: **AI 競合サマリ** — Insights データを自然言語で要約
- 非 AI: **新着動画通知** — 日次ジョブで Watchlist 新規投稿検出、メール / 履歴通知
- 非 AI: **チャンネル成長軌跡** — 長期推移の推定可視化

## 3. KPI 仕様

### 3.1 チャンネル KPI（8 指標）

| # | 指標名 | 計算式 / 取得元 |
|---|---|---|
| 1 | チャンネル名 | `channels.snippet.title` |
| 2 | 登録者数 | `channels.statistics.subscriberCount` |
| 3 | 動画本数 | `channels.statistics.videoCount` |
| 4 | 総再生数 | `channels.statistics.viewCount` |
| 5 | 平均再生数 | 総再生数 ÷ 動画本数 |
| 6 | 平均伸び率 | 期間内動画の伸び率の平均 |
| 7 | 動画投稿頻度 | 期間内動画数 ÷ 期間日数（本/日） |
| 8 | チャンネル作成日 | `channels.snippet.publishedAt` |

### 3.2 動画 KPI（6 指標 + 拡張 3 指標）

| # | 指標名 | 計算式 / 取得元 |
|---|---|---|
| 1 | 公開日 | `videos.snippet.publishedAt`（YYYY-MM-DD） |
| 2 | タイトル | `videos.snippet.title` |
| 3 | 再生時間 | `videos.contentDetails.duration`（ISO 8601 → MM:SS / H:MM:SS） |
| 4 | 再生数 | `videos.statistics.viewCount` |
| 5 | 高評価数 | `videos.statistics.likeCount` |
| 6 | 伸び率 | `viewCount ÷ subscriberCount × 100`（小数第 2 位） |
| **7** | **高評価率** | `likeCount ÷ viewCount × 100`（小数第 2 位） |
| **8** | **コメント率** | `commentCount ÷ viewCount × 100`（小数第 2 位） |
| **9** | **バイラル係数** | `伸び率 × 高評価率 ÷ 100` |

### 3.3 伸び率の色分け閾値

| 範囲 | 表示色（ダークテーマ基準） | 意味 |
|---|---|---|
| **100% 以上** | ライムグリーン | **勝ち動画（ヒット）** |
| 30% 以上 100% 未満 | アンバー | 健全 |
| 30% 未満 | ニュートラルグレー | 平均以下 |

### 3.4 推移グラフ仕様

| 項目 | 内容 |
|---|---|
| 集計単位 | 月単位（最大 12 ヶ月分） |
| グラフ 1 | 月別投稿本数（棒グラフ） |
| グラフ 2 | 月別総再生数（折れ線グラフ） |
| 比較画面 | 3 チャンネル分の折れ線を重ね描画（線色で識別） |

### 3.5 投稿時間ヒートマップ仕様

| 項目 | 内容 |
|---|---|
| マトリクス | 7 (曜日) × 24 (時) |
| セル値モード A | 投稿数（公開日時の累積） |
| セル値モード B | 平均伸び率（伸び率の集計） |
| タイムゾーン | JST 固定（YouTube API は UTC 返却なので変換） |

## 4. CSV 出力仕様

### 4.1 列定義（単一チャンネル: 12 列）

| # | 列名 | 内容 | 形式 |

| 5 | 動画URL | YouTube URL | https://youtube.com/watch?v=... |
| 6 | 再生数 | 視聴回数 | 整数 |
| 7 | 高評価 | いいね数 | 整数 |
| 8 | 伸び率(%) | 伸び率 | 小数 |
| 9 | コメント数 | コメント数 | 整数 |

## 6. 非機能要件

### 6.1 性能要件

| 項目 | 目標値 |
|---|---|
| 検索結果初回表示 | ≤ 5 秒（中規模チャンネル：動画 100 本程度） |
| 検索結果初回表示（大規模） | ≤ 15 秒（動画 1,000 本程度） |
| Watchlist 一括更新（30 ch） | ≤ 5 分（バックグラウンドジョブ + 進捗表示） |
| Insights ダッシュボード表示 | ≤ 3 秒（AnalysisCache 利用時） |
| キーワード市場分析実行 | ≤ 10 秒 |
| ページ遷移 | ≤ 500 ms |
| 同時接続ユーザー数 | 100（MVP 時点） |
| CSV エクスポート | 1,000 行で ≤ 3 秒 |

### 6.2 可用性要件

| 項目 | 目標値 |
|---|---|
| 稼働率 | 99.5%/月（Vercel + Neon の SLA に準拠） |
| 計画停止 | 月 1 回まで、事前告知 |
| バックアップ | Neon の自動バックアップ（7 日保持） |

### 6.3 セキュリティ要件

| 項目 | 内容 |
|---|---|
| 通信 | HTTPS 必須（HTTP → HTTPS リダイレクト） |
| パスワード | bcrypt cost 12 でハッシュ化 |
| API キー（YouTube + AI） | AES-256-GCM で暗号化、サーバー外に出さない |
| Cookie | `HttpOnly`, `Secure`, `SameSite=Lax` |
| CSRF 対策 | NextAuth.js 標準 + Server Actions の CSRF Token |
| XSS 対策 | React の自動エスケープ + CSP ヘッダ |
| SQL Injection | Prisma パラメータバインディング |
| Rate Limit | 認証エンドポイント 10 req/min/IP、検索 30 req/min/user、キーワード分析 5 req/min/user |
| 監査ログ | 認証イベント・APIキー操作・Watchlist 操作をログ保存（90 日） |

### 6.4 ユーザビリティ要件
- 主要操作（検索開始、Watchlist 追加、CSV DL）は **2 クリック以内**
- エラー時は次のアクションを明示
- 初回ユーザー向けオンボーディング: API キー登録 → Watchlist 追加 → Insights の 3 ステップツアー
- Quota 高消費操作は事前確認モーダル必須

### 6.5 保守性要件
- TypeScript strict mode、ESLint エラーゼロを CI でブロック
- 単体テストカバレッジ：KPI 計算・Insights 集計ロジック 90% 以上
- E2E テスト：主要 6 導線（サインアップ → キー登録 → 検索 → Watchlist → Insights → CSV）

### 6.6 互換性要件

|---|---|
| Chrome | 最新 2 バージョン |
| Edge | 最新 2 バージョン |
| Safari | 最新 2 バージョン |
| Firefox | 最新 2 バージョン |
| モバイル Safari | iOS 16 以降 |
| Chrome (Android) | 最新 2 バージョン |

### 6.7 法令遵守要件
- YouTube API Services Terms of Service 準拠
- 各生成 AI ベンダー利用規約準拠
- 個人情報保護法対応（プライバシーポリシー、利用規約整備）

## 7. デザイン要件

オリジナル「つべサーチ」（ライトブルー基調・カード並列のシンプル SaaS）とは **明確に異なる方向性** を採用。

| 要素 | 方針 |
|---|---|
| テーマ | ダーク基調デフォルト、ライトはオプション |
| カラーパレット | ニュートラルグレー基盤 + アクセント 1 色（ライム/シアン系） |
| レイアウト | 左サイドバー固定 + メインダッシュボード（カード並列ではなくテーブル中心） |
| タイポグラフィ | UI: Inter / Geist Sans、数値: JetBrains Mono / Geist Mono |
| 情報密度 | 高（パワーユーザー向け） |
| データ表現 | スパークライン・インラインミニチャート・数値プログレスバーを多用 |
| アニメーション | 抑制的 |

詳細は [03_basic_design.md §2 画面設計](./03_basic_design.md) を参照。

## 8. 受け入れ基準（MVP）

以下すべてを満たした時点で MVP リリース可とする。

### 基本機能
- [ ] サインアップ → メール認証 → ログインが完了する
- [ ] `/settings` で YouTube + AI 3 ベンダーの API キーを登録・更新・削除できる
- [ ] `/search` で URL を入力するだけで分析が開始される（キー設定 UI が検索画面に存在しない）
- [ ] `/search` で KPI と推移グラフが表示される
- [ ] `/compare` で 3 チャンネル同時分析ができる
- [ ] CSV / ZIP がダウンロードできる

### 横断・市場分析機能
- [ ] `/watchlist` で 30 ch まで追加・削除・タグ付けできる
- [ ] 検索 → 「★ Watchlist 追加」→ `/watchlist` で確認、の流れが動作
- [ ] `/insights` で勝ち動画パターン・ヒートマップ・サムネギャラリー・頻出語・尺分布が表示される
- [ ] `/keywords` でキーワード分析が動作し、Quota 警告モーダルが事前表示される
- [ ] `/discover` で類似チャンネル推薦が動作し、Watchlist 追加が 1 クリックで可能

### 運用品質
- [ ] `/history` から過去の検索・比較・キーワード分析・競合発見を再実行できる
- [ ] モバイル（iPhone 13 サイズ）で操作可能
- [ ] デザインが「つべサーチ」と明確に異なる（ダーク基調 / サイドバー / ライム系アクセント）
- [ ] 主要 6 導線の E2E テストが通る
- [ ] Lighthouse スコア：Performance ≥ 80, Accessibility ≥ 90, Best Practices ≥ 90
